Container

public protocol Container

Container protocol should be implemented by Factories that produce any type of the view controllers that can be considered as Containers (eg: UINavigationController, UITabBarController, etc)

Container apply one merged action and then populate a full stack of view controllers that was built by the associated factories in one go. Example: Steps require to populate n view controllers in UINavigationController stack and it can do so. Merge action implementation is mandatory for any actions and should be implemented if it can be done.

  • merge(_:)

    Receives an array of factories whose view controllers should be merged into current container factory before it actually builds a container view controller with child view controllers inside. Example: UINavigationController with N pushed into it UIViewControllers.

    Example: UINavigationController as a container expects push action of any kind. If a factory from factories array contains one with a present modally action, it will be returned back as an unsupported one.

    Array of factories to be handled by container factory.

    Return Value

    Array of factories that are not supported by this container type. Router should decide how to deal with them.
